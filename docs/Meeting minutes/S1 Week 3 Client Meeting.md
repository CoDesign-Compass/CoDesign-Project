# S1 Week 3 Client Meeting
## Date 
12/03/2026, 1:00 pm --1:40 pm

## Loaction
The Hive, Building 108, ANU
## Participants
Ricky Chen, Jingwen Zhang, Yu Pan, Jiaxin Li, Aria Fung, Zhimeng Zhu, Shuo Li, Heidi Prowse (client)
## Goals
- Demonstrate the current prototype of the CoDesign Compass platform.
- Collect client feedback on user interaction flow and usability.
- Identify improvements to enhance accessibility and clarity for users.
- Discuss deployment and domain considerations for the platform.

## Discussion Topics
| Time | Item | Presenter | Notes |
| ---- | --------- | ----- | ---- |
| 13:00 | Login and Registration Flow | Client & Team | The client noted that the current system only supports **login**, without a visible **registration option**. The client suggested adding a **Register button** on the homepage so new users can register before logging in. This allows users to clearly choose between **Login** and **Register** when entering the platform. |
| 13:05 | User Journey and Accessibility | Client | The client emphasized that many users may arrive via an **email invitation link** and may not understand whether they need to **start directly, log in, or register**. The team should simplify the **user journey** and ensure the steps are clear and intuitive. |
| 13:10 | Automatic Email Notification            | Client        | The client suggested adding a feature that automatically **sends an email notification to users who have previously completed a questionnaire** when a **new issue is created**. This would help re-engage previous participants and encourage continued feedback. |
| 13:15 | Why / How Question Layout               | Client        | The client suggested adjusting the layout of the **Why** and **How** pages. Currently, questions remain fixed at the top of the page. The client recommended moving each question **directly above the answer input field** so users can easily read the question while responding. |
| 13:20 | User Testing                            | Client        | The client recommended testing the system with people unfamiliar with the project to see whether the workflow is intuitive. This would help validate the usability of the platform from a real user perspective. |
| 13:25 | Deployment and Domain Name              | Team & Client | The team explained that the current system is deployed using a **free hosting service**. The client suggested that using a **clear domain name (e.g., codesigncompass)** may increase user trust and reduce the chance of emails being marked as spam. |
| 13:30 | Client Availability for Future Meetings | Client        | The client mentioned she plans to work at **The Hive on Thursdays regularly**, making it easier for the team to demonstrate progress and discuss updates during upcoming sprints. |

## Decisions
|Decision|Outcome|
|---|---|
|Add a **Register button** on the homepage|Agreed|
|Implement **automatic email notifications** when new issues are created|Agreed|
|Adjust **Why / How page layout** to display questions directly above answer fields|Agreed|
|Conduct usability testing with external users|Recommended|
|Consider purchasing a **custom domain name** for the deployed platform|Suggested|
